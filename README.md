# 일식·월식 시뮬레이션 ☀️🌍🌑

달은 일식과 월식에서 모두 **동쪽(왼쪽)으로 공전**합니다. 그런데 일식에서는 태양이 **오른쪽(서)부터**,
월식에서는 달이 **왼쪽(동)부터** 가려져 **반대로** 보입니다. 학생들의 질문에서 출발해 그 까닭을
순수 HTML/Canvas 시뮬레이션으로 풀어봅니다. (빌드 도구·외부 라이브러리 없음)

**구성**
- **첫 페이지 `index.html`** — 일식·월식에 대한 두 학생 질문 + 두 버튼
- **일식 `solar.html` + `solar.js`** — 달그림자 속도(각속도 vs 선속도), 태양은 오른쪽(서)부터
- **월식 `lunar.html` + `lunar.js`** — 달이 지구 그림자를 통과, 달은 왼쪽(동)부터
- 공통 스타일 `styles.css`

> **라이브 데모:** `https://devrobertson.github.io/solar-eclipse-speed/` (배포는 저장소 소유자가 구성)

## 🌞 일식 — 무엇을 보여주나

- **학생 질문:** “태양은 자전으로 1시간에 **15°**, 달은 공전(0.5°/h)이 빠져 **14.5°/h**로 조금 더 느리게
  서쪽으로 간다. 달이 더 느리니 태양을 **왼쪽부터** 가리는 것 아닐까?” (오개념)
- **답(방향):** *겉보기 상대운동*이 결정 — 달이 태양 대비 **동쪽으로 0.5°/h** 이동 → 태양 **오른쪽(서)부터**.
- **답(속도):** 그림자가 **지표를 지나가는 속도**는 *선속도*(v=ω·r)가 결정 — 달이 60배 멀어 1,023 m/s로 지표(465)의 **2.2배**.

| | 각속도 ω | 선속도 v = ω·r |
|---|---|---|
| 지구 자전(지표) | ≈15°/h | ω·R<sub>지구</sub> ≈ **465 m/s** (≈1,673 km/h) |
| 달 공전 | ≈0.5°/h | ω·d<sub>달</sub> ≈ **1,023 m/s** (≈3,684 km/h) |
| 비교 | 지구가 **27.4×** | 달이 **2.2×** |

상수: R<sub>지구</sub>=6,371 km, d<sub>달</sub>=384,400 km, 항성일 23.934 h, 항성월 27.32 일.
패널: ① 북극 상공 ② 관측자 하늘(오른쪽부터) ③ 지표 위 선속도 경주 ④ 각속도 vs 선속도.

## 🌑 월식 — 왜 반대 방향인가

- **학생 질문:** “일식 때는 달그림자가 지표보다 빠르게 **동쪽으로** 가는데, 월식은 왜 **반대 방향**으로 보일까?”
- **답:** 둘 다 달은 동쪽으로 공전하지만, **일식은 달이 *가리는 쪽*** (태양 서쪽부터),
  **월식은 달이 *가려지는 쪽*** — 앞서가는 **동쪽(왼쪽) 가장자리**가 지구 그림자에 먼저 들어가 **달 왼쪽(동)부터** 어두워진다.
  → 가려지기 시작하는 쪽이 서로 반대(오른쪽 ↔ 왼쪽).
- 패널: ① 북극 상공(지구 본영 통과) ② 관측자 하늘(밤, 왼쪽부터) ③ 일식 ↔ 월식 비교 ④ 요약.

**공통 컨트롤:** 재생/일시정지 · 시간 슬라이더 · 속도(0.25–4×) · 스냅샷(1차 접촉 → 최대 → 종료) · 벡터/방향 토글.

## 가정 & 한계

- 적도·정오 근처, 평행한 햇빛(태양 무한 원거리), 항성일/항성월 기준의 **단순화 모형**입니다.
- 화면의 거리·크기는 보기 좋게 과장했지만, **각속도·선속도의 비율은 실제 값**을 사용했습니다.
- 방향(좌·우)은 **북반구에서 남쪽 하늘을 볼 때**(동=왼쪽, 서=오른쪽) 기준입니다.
- 달 궤도 경사·이심률, 위도에 따른 그림자 속도 변화 등은 의도적으로 생략했습니다.
